Keys

A key in a Database Management System (DBMS) is an attribute or set of attributes that uniquely identifies a row or tuple in a table or relation. Keys are essential to relational databases and are used to establish relationships between tables and columns. They also allow rows to be separated and used for calculations. There are different types of keys used in a database, such as:

Primary Key

A primary key, as suggested by the name, is the main key used as the identifier in a database management system to uniquely identify each record in the database. The primary key has to have a unique value and not be NULL.

Candidae key

A candidate key is a field or a combination of fields which can be used to uniquely identify the records in the database. One of the field present in the candidate key set is chosen as the primary key in the database. If none of the candidate keys are chosen as the primary key, then all of them are taken as the alternatte key.

Alternate Key

An alternate key is the candidae key which is not chosen as the primary key for the database. In the case where there are more than two candidate keys, only one of them is chosen as the primary key even though the remaining candidate keys ( or the alternate keys ) can also uniquely identify the records of the database.

Foreign Key

When two databases are combined together into a single databse, the primary key of one database is taken as the Foreign Key which is used in order to identify the records of that database in the newly fromed database.

Super Key

A super key is a combination of keys that act as an identifier for a database even if the uniqueness of the combination is not certain.
